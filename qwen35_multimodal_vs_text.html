Qwen3.5 · 多模态 vs 纯文本请求处理差异

基于 vLLM + vllm-ascend 源码分析 · 2026-05-21

核心问题：图片+文字请求用的是同一个语言模型吗？

结论：是同一个语言模型。没有专门处理多模态的第二个独立 LLM。视觉编码器（Visual Encoder）只负责把图片像素转换成 embedding 向量，然后插入文本 embedding 序列，最终统一送入 同一个 Qwen3 语言模型。

总结：Qwen3.5 是典型的 "视觉编码器 + 语言模型" 融合架构（类似 LLaVA 系列）。视觉编码器和语言模型权重独立，但推理时共用同一套 Transformer layers 完成最终的理解和生成。Decode 阶段两种请求路径完全一致。

视觉编码器输入/输出

patch_embed 输出
shape: (num_patches, embed_dim)
embed_dim = 1152（默认）
像素 patch → embedding。
Conv3d 卷积提取 patch 特征。
Vision Transformer 输出
Qwen2-VL: (num_patches, 1152)
Qwen3-VL: (num_patches, 1152×(1+deepstack))
所有 Vision Transformer block 处理后的特征。
Qwen3 DeepStack 拼接了多个中间层特征。
Merger（合并后）输出
shape: (num_vision_tokens, hidden_size)
num_vision_tokens = num_patches ÷ merge_size²
空间合并后的视觉 embedding。
merge_size=2 → 每 2×2 个 patch 合并为 1 个 token。
这是插入 LLM 序列的实际视觉向量。

shape: (retained_tokens, hidden_size + 5)
